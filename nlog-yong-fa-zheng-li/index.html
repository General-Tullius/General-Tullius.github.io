NLog用法整理

2021-02-01 3 min read # dotnet

NLog是一个简单灵活的.NET日志记录类库。

官方文档 Nlog

使用流程

添加相关包引用

NLog.Web.AspNetCore

NLog

添加nlog.config

配置内容如下

<?xml version="1.0" encoding="utf-8" ?>
<nlog xmlns="http://www.nlog-project.org/schemas/NLog.xsd"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <targets>
    <target name="logFile" xsi:type="File" fileName="logs/log-${shortdate}.log" layout="[${longdate}]：${message}" />    
    <target name="errFile" xsi:type="File" fileName="logs/err-${shortdate}.log" layout="[${longdate}]：${message}" />    
  </targets>
  <rules>
    <logger name="*" level="Info" writeTo="logFile" />
    <logger name="*" level="Error" writeTo="errFile" />
  </rules>
</nlog>

修改program.cs

public static IHostBuilder CreateHostBuilder(string[] args) =>
            Host.CreateDefaultBuilder(args)
                .ConfigureWebHostDefaults(webBuilder =>
                {
                    webBuilder.UseStartup<Startup>();
                })
                .ConfigureLogging(logging => { logging.ClearProviders(); logging.AddNLog("config/nlog.config"); });

记录日志

namespace Example.Contollers
{
    [Route("api/[controller]")]
    [ApiController]
    public class ExampleController : ControllerBase
    {
        readonly ILogger<ExampleController> _logger;
        
        public ExampleController(ILogger<ExampleController> logger)
        {
            _logger = logger;
        }
    }
    
    [HttpPost("")]
    public ActionResult Post([FromBody] EType value)
    {
        _logger.LogInformation("日志内容");
        return Ok();
    }
}

其它说明

关于配置文件说明

targets部分

  • targets的作用

    targets用来设定日志记录相关的一些规则，例如日志存放位置、日志文件名称、日志内容记录格式等。

  • 属性说明

    name相当于id，用于选定某一个日志创建规则

    type日志记录类型，可以是文件File、数据库Database或者邮件Mail

    fileName用于设定日志文件存放路径和文件名称

    layout用于设定日志内容记录的格式

  • 示例

    <targets>
    <target name="logFile" xsi:type="File" fileName="logs/log-${shortdate}.log" layout="[${longdate}]：${message}" />    
    <target name="errFile" xsi:type="File" fileName="logs/err-${shortdate}.log" layout="[${longdate}]：${message}" />    
</targets>

rules部分

  • rules的作用

    rules用来设定某处产生的某级别的日志应该采用哪个日志配置来记录。

  • 属性说明

    writeTo用来选定tagets中的某一个配置，取值为target的名字

    name用来筛选产生日志的地方（命名空间、类等）

    level用来控制某一特定的日志级别，还可以用minlevel、maxlevel、levels等属性来选定多个日志级别

  • 示例

    <rules>
    <logger name="Example.*" level="Info" writeTo="logFile" />
    <logger name="Example.*" level="Error" writeTo="errFile" />
</rules>
